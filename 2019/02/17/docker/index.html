docker的入门

  • 2019-02-17

一.Docker的概念的定义

      Docker是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到任何流行的机器上，通过仓库来集中管理镜像。

二.docker中的重要概念

  2.1镜像

       镜像就相当与一系列文件，可以包括应用程序的文件、配置文件、运行环境的文件。结果如图2-1。最底端是一个文件引导系统，即bootfs。Docker用户不会与引导文件系统有直接的交互。Docker镜像的第二层是root文件系统rootfs，通常是一种或多种操作系统，例如ubuntu等。在Docker中，文件系统永远都是只读的，在每次修改时，都是进行拷贝叠加从而形成最终的文件系统。Docker称这样的文件为镜像。一个镜像可以迭代在另一个镜像的顶部。位于下方的镜像称之为父镜像，最底层的镜像称之为基础镜像。

  2.2容器

      当从一个镜像启动容器时，Docker会在最顶层加载一个读写文件系统作为容器。当需要进行对可读层进行写的时候，将会将镜像文件复制到容器中进行操作，因此一个镜像可以对应多个容器做到互不影响。

  2.3仓库

      仓库有两种，分别是中央仓库与本地仓库，中央仓库存储了各种镜像，可以通过命令将镜像拉取到本地仓库中。

三.docker中的基本命令

1.拉取镜像 

docker pull hello-world

2.查看镜像 

docker images

如图下图，可以看到镜像相关的信息

3.运行nginx镜像

docker run -d hub.c.163.com/library/nginx

4.查看容器信息

docker ps

如下图，可以查看运行镜像生成容器的信息，比如创建时间、唯一的ID、端口映射等信息。

5.进入容器（ngnix运行在linux容器上,进入linux界面）

docker exec -it 69e(容器ID）bash

6.退出容器 

exit

7.停止容器 

docker stop 69e

8.docker网络（默认使用bridge模式）

docker run -d -p 8080:80 hub.c.163.com/library/nginx  指定主机与容器映射端口
docker run -d -P  hub.c.163.com/library/nginx  主机上随机开放一个端口与容器端口进行映射

五.使用开源组件JPRESS构建个人博客系统

1.从git上下载JPRESS的war包

2.基于tomcat制作自己的镜像
    首先生成一个Dockerfile文件,告诉docker怎么制作镜像，制作镜像得每一步是怎么样的，使用docker built执行dockerfile中描述的事件并构建出来自己的镜像。 如下为dockerfile的代码

from hub.c.163.com/library/tomcat  #以tomcat镜像为起点的
MAINTAINER ztaocoder xxx163@.com #个人信息
copy jpress.war /user/local/tomcat/webapps #将war包复制到tomcat的webapps目录下

最后执行 docker built -t jpress:latest . 命令

3.安装mysql数据库
     首先从中央仓库（网易云镜像仓库）拉取镜像

docker pull hub.c.163.com/library/mysql:latest

     然后启动镜像，设置相应的登陆密码与数据库名字。

docker run -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=000000  -e MYSQL_DATABASE=jpess hub.c.163.com/library/mysql

4.启动jpress镜像

docker run -d -p 8080:8080 jpress

5.根据jpress的指导连接数据库，建立个人博客
